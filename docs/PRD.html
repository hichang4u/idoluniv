01

제품 개요 Product Overview

Executive Summary
IdolUniv는 K-pop을 중심으로 한 글로벌 아이돌 팬덤 커뮤니티 플랫폼입니다. 단순 게시판을 넘어 팬들이 직접 콘텐츠를 생산·소비·유통하며 아이돌과 실질적으로 연결되는 슈퍼앱(Super App) 형태의 차세대 팬덤 생태계를 지향합니다.

🎤 제품명

IdolUniv (아이돌유니브)

🏢 운영 주체

1인 운영 → 커뮤니티 자치 구조로 점진 전환

🌍 서비스 범위

글로벌 (한/영/일/중/태 다국어 지원)

🖥️ 플랫폼

Web (Mobile First) + PWA

☁️ 인프라

Vercel (Frontend) + Supabase (Backend)

📅 MVP 목표

2026 Q2 출시

문제 정의 (Problem Statement)

😤 현재 팬들의 불편함

  • • Weverse, Bubble, X, Instagram에 분산된 팬 활동
  • • 언어 장벽으로 인한 글로벌 팬덤 소통 어려움
  • • 팬 창작물 수익화 채널 부재
  • • 컴백·콘서트 일정 통합 관리 부재
  • • 팬덤 기여도 인정 시스템 없음

✅ IdolUniv의 해결책

  • • 모든 팬 활동을 하나의 플랫폼으로 통합
  • • AI 기반 실시간 다국어 번역
  • • 팬 마켓 + 후원 + 펀딩 생태계
  • • 통합 팬 캘린더
  • • 팬덤 레벨 시스템 & 배지

기능 요구사항 Feature Requirements

1단계 · MVP
Foundation — 핵심 커뮤니티 기능으로 초기 팬덤 확보
2026 Q2
  • ✓ 소셜 로그인 (Google, Kakao, Twitter/X)
  • ✓ 아이돌 그룹별 전용 게시판 (텍스트, 이미지, 영상)
  • ✓ 댓글 & 대댓글, 좋아요, 스크랩
  • ✓ 실시간 채팅방 (그룹별 라운지)
  • ✓ 반응형 UI (Mobile First)
  • ✓ 관리자 대시보드 (콘텐츠 관리, 신고 처리)
  • ✓ SEO 최적화 (Next.js App Router + SSR/SSG)
  • ✓ 회원 프로필 & 팬 프로필 설정
  • ✓ 이미지 업로드 (Supabase Storage)
  • ✓ 콘텐츠 신고 및 차단 시스템
2단계 · 성장
Engagement — 팬 참여도와 체류 시간 극대화
2026 Q3~Q4
  • ○ 팬덤 레벨 시스템 (활동량 기반 배지·칭호·권한)
  • ○ 투표 & 실시간 차트 (인기 순위, 팬 총선)
  • ○ 팬 캘린더 (컴백, 콘서트, 팬사인회 일정)
  • ○ 멀티미디어 피드 (숏폼 영상 + 포토카드 갤러리)
  • ○ 다국어 지원 (한/영/일/중/태 AI 번역)
  • ○ 푸시 알림 (신규 콘텐츠, 댓글 답글 실시간)
  • ○ 검색 고도화 (아이돌명, 해시태그, 통합 검색)
  • ○ 인플루언서 팬계정 인증 시스템
  • ○ 다크/라이트 테마 전환
  • ○ 게시물 예약 발행
3단계 · 수익화
Creator Economy — 팬이 창작자가 되는 생태계
2027 Q1~Q2
  • ○ 팬 창작 마켓 (팬아트, 굿즈 디자인, 팬픽 판매)
  • ○ 후원 & 펀딩 (생일 카페, 광고판 크라우드펀딩)
  • ○ 구독 멤버십 (프리미엄 채널, 독점 콘텐츠, 광고 제거)
  • ○ 포인트 이코노미 (활동 포인트 → 굿즈 교환)
  • ○ 아이돌 공식 연동 (소속사 API 파트너십)
  • ○ 결제 연동 (Toss Payments + Stripe)
  • ○ 크리에이터 정산 대시보드
  • ○ 팬 굿즈 주문 제작 + 배송 연동
4단계 · 미래
AI & Web3 — 차세대 팬덤 기술 선도
2027 Q4+
  • ○ AI 팬픽 생성 (Claude 기반 팬덤 소설 자동 생성)
  • ○ AI 포토카드 편집 (필터, 레이아웃, 레트로 효과)
  • ○ AI 챗봇 "팬마스터" (공지, Q&A, 규칙 안내)
  • ○ 팬 NFT 카드 (디지털 포토카드 소장·거래)
  • ○ 팬 토큰 이코노미 (기여 기반 토큰 → 투표권)
  • ○ AR 포토 부스 (아이돌과 가상 인증샷 WebAR)
  • ○ 가상 팬미팅 공간 (3D 메타버스 이벤트)
  • ○ OpenAI Whisper 기반 영상 자막 자동 생성

  • • LCP < 2.5초 (Core Web Vitals)
  • • FID < 100ms
  • • CLS < 0.1
  • • API 응답 < 200ms (P95)
  • • 동시 접속 10,000명 처리

  • • JWT 기반 인증 (7일 만료)
  • • Rate Limiting (API별)
  • • 업로드 파일 바이러스 스캔
  • • OWASP Top 10 대응

📱 접근성

  • • WCAG 2.1 AA 수준 준수
  • • 키보드 내비게이션 지원
  • • 스크린 리더 호환
  • • 색상 대비 4.5:1 이상
  • • 텍스트 크기 조절 지원

🔄 가용성

  • • SLA 99.9% (월 다운타임 < 43분)
  • • Vercel Edge Network CDN
  • • Supabase 자동 일일 백업

📈 확장성

  • • Edge Functions (글로벌 배포)
  • • 이미지 자동 최적화 (Vercel)
  • • 데이터 파티셔닝 설계
  • • CDN 캐싱 전략

🌐 국제화

  • • i18n: 한/영/일/중/태
  • • RTL 언어 대응 설계
  • • 타임존 자동 감지
  • • 통화 현지화 (KRW/USD/JPY)
  • • 문화권별 날짜 형식

UX 원칙 UX Principles

# 원칙 정의 측정 지표
01 Mobile First 팬들의 90%+ 모바일 접근 — 앱 수준의 모바일 경험 제공 모바일 Lighthouse 점수 90+
02 3-Tap Rule 원하는 콘텐츠까지 3번 이내 탭으로 도달 평균 탐색 depth ≤ 3
03 Dark Mode 기본 Z세대 선호 + 눈 피로도 감소 — 다크 테마를 기본으로 설정 다크 모드 사용률 70%+
04 초고속 로딩 Core Web Vitals LCP 2.5초 이하 목표 LCP < 2.5s, FID < 100ms
05 접근성(a11y) WCAG 2.1 AA 수준 준수 — 모든 사용자 포용 axe-core 위반 0건
06 팬 감성 디자인 아이돌 세계관에 어울리는 감성적 비주얼 아이덴티티 NPS 50+ 목표

[CDN / Edge Network — Vercel Global] │ [User Browser (Mobile/Web)] ──HTTPS──► [Vercel Edge Runtime] │ ┌───────────────────────┼───────────────────────┐ │ │ │ [Next.js SSR/SSG] [API Routes] [Edge Functions] (App Router 14) (서버리스) (전처리·AI 호출) │ │ │ └───────────────────────┼───────────────────────┘ │ [Supabase BaaS] ┌──────────────────┼──────────────────┐ │ │ │ [PostgreSQL] [Realtime] [Storage] [Auth/RLS] (채팅·알림·투표) (이미지·영상) │ │ [외부 AI 서비스] │ ┌───────────────────────┐ └─────►│ Claude API (팬픽·번역) │ │ Replicate (이미지 생성) │ │ Whisper (영상 자막) │ └───────────────────────┘ [결제] [모니터링] [CI/CD] Toss Payments Sentry (에러) GitHub Actions Stripe (글로벌) Vercel Analytics Vercel Preview

데이터 흐름

📝 콘텐츠 작성

사용자 → Next.js → Supabase Edge Function → PostgreSQL → Realtime 브로드캐스트

💬 실시간 채팅

사용자 → Supabase Realtime WebSocket → 채팅방 구독자 브로드캐스트

🤖 AI 기능

사용자 요청 → Edge Function → Claude/Replicate API → 결과 반환 + Storage 저장

💳 결제

사용자 → Toss/Stripe → Webhook → Edge Function → DB 업데이트 → 권한 부여

리스크 관리 Risk Management

HIGH
저작권 분쟁 — 팬 창작물 내 아이돌 이미지 사용

완화: 명확한 저작권 정책 고지, 소속사 클레임 접수 채널 운영, DMCA 준수 자동화. 상업적 사용 구분 가이드라인 수립.

HIGH
1인 운영 한계 — 트래픽 급증 시 대응 불가

완화: 자동화 인프라 우선 설계 (AI 콘텐츠 필터, 자동 신고 처리), 커뮤니티 모더레이터 조기 구성, Vercel/Supabase 자동 스케일.

MEDIUM
Weverse/카카오 등 대형 플랫폼의 경쟁 기능 출시

완화: 팬 주도 생태계 차별화, AI 기능 선점, 커뮤니티 자치 구조로 플랫폼 락인. 소속사 직접 의존도 최소화.

MEDIUM
악성 콘텐츠 및 스팸 계정 증가

완화: AI 1차 필터 + Supabase RLS 자동 차단, Rate Limiting 전체 적용, 커뮤니티 신고 기반 빠른 처리.

MEDIUM
AI API 비용 급증 (Claude, Replicate)

완화: 프리미엄 멤버십 AI 기능 분리, 사용량 한도 설정, 캐싱 전략 적극 활용. 사용량 기반 비용 모니터링 알림.

LOW
Supabase / Vercel 서비스 장애

완화: 자동 일일 백업, 상태 페이지 모니터링, Sentry 에러 알림, 장애 공지 절차 사전 수립.
